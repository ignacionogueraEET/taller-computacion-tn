Actividad integradora: Registro y análisis de componentes y mediciones electrónicas

Contexto (rol profesional)

Un laboratorio de electrónica trabaja con distintos circuitos y prototipos. Con frecuencia se utilizan componentes como resistencias, capacitores, diodos y circuitos integrados, además de realizar mediciones con instrumentos.

El problema actual es que:

  • No hay registro ordenado de componentes disponibles
  • Las mediciones realizadas en prácticas no se sistematizan
  • Es difícil detectar fallas repetidas en circuitos o componentes

El laboratorio solicita una herramienta que permita organizar información técnica y facilitar el análisis de componentes y mediciones.

Consigna

Deberás desarrollar una solución utilizando una planilla de cálculo que permita gestionar información del laboratorio de electrónica.

⚠️ Importante: No se indicarán herramientas específicas. Se evaluará tu capacidad de organizar datos técnicos y construir una solución funcional.

Requerimientos del laboratorio

1. Registro de componentes

La herramienta debe permitir:

  • Identificar cada componente (tipo y código o referencia)
  • Clasificar por tipo (resistencia, capacitor, diodo, transistor, etc.)
  • Registrar características relevantes (valor, tolerancia, estado, cantidad disponible)

2. Registro de prácticas o mediciones

Debe permitir:

  • Registrar prácticas realizadas en el laboratorio
  • Incluir fecha, circuito o práctica, y componentes utilizados
  • Registrar mediciones relevantes (por ejemplo: voltajes, corrientes, resistencias medidas)

3. Control y análisis

La herramienta debe facilitar:

  • Identificar componentes más utilizados o más críticos
  • Detectar componentes defectuosos o con fallas recurrentes
  • Analizar tendencias en mediciones o resultados de prácticas

4. Organización de la información

  • Mantener una estructura clara para consultas rápidas
  • Evitar duplicación o pérdida de datos
  • Facilitar la actualización del inventario

5. Visualización

  • Incluir al menos una representación gráfica útil (por ejemplo: uso de componentes, fallas registradas, o distribución por tipo)

Decisiones técnicas (parte central)

Deberás definir:

  • Cómo organizarás el inventario y los registros
  • Qué criterios usarás para clasificar la información
  • Cómo automatizarás el análisis de datos

🔼 Desafío opcional (nivel avanzado)

El laboratorio quiere mejorar la detección de problemas antes de que afecten las prácticas.

Tu herramienta deberá permitir:

  • Identificar componentes con mayor probabilidad de falla (según historial o uso)
  • Detectar patrones en mediciones que indiquen posibles errores de montaje o fallas de componentes
  • Proponer algún criterio de "alerta" para componentes críticos

💡 Este punto implica pasar de registrar datos a interpretarlos para anticipar problemas técnicos.
